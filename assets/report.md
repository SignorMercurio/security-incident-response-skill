## 响应行动 / Response Actions

<!-- 每个阶段保留最重要的 3-5 条；删掉不适用的通用建议。 -->

::: phase 抑制阶段 / Containment
抑制阶段只写阻断攻击者继续操作的动作。状态标记含义：[x] 已完成，[/] 进行中，[ ] 未开始。

- [x] 已完成：[隔离/封禁/吊销/阻断等关键动作]
- [/] 进行中：[动作、责任方、卡点或预计完成时间]
- [ ] 未开始：[需要客户确认的止损动作]
:::

::: phase 根除阶段 / Eradication
根除阶段只写遗留能力和根因条件的清理状态。不要把建议写成事实。

- [x] 已完成：[恶意文件/后门账号/异常服务/云侧策略清理]
- [/] 进行中：[补丁修复/密钥轮转/权限回收/漏洞修复]
- [ ] 未开始：[需要客户确认的根因修复]
:::

::: phase 恢复阶段 · 处置建议 / Recovery & Hardening
恢复阶段只写本事件最相关的长期加固建议，按风险优先级排序，不堆通用安全清单。

- [ ] 身份与权限：[本事件相关的权限收敛、白名单、密钥轮转]
- [ ] 暴露面：[需要关闭、加白或接入防护的入口]
- [ ] 漏洞与配置：[需要升级、修复或复查的同类系统]
- [ ] 可观测性：[必须补开的日志和留存周期]
- [ ] 演练与治理：[只保留与复发风险直接相关的治理动作]
:::
